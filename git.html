что такое git workflow?

git workflow представляет собой один единый проект с ветками основной разработки

для чего нужен git workflow?

git workflow предоставляет пользователю удобную работу с проектом тем что имеет куча отдельных веток от основной ветки main

как работает git workflow?

пользователь создает отдельную ветку если захочет добавить что-то новое в свой проект, это может быть как просто создание нового обновления, либо добавление фичи, и пользовательь позже сам решает добавлять ли эти ветки в освноную ветку main

что такое git rebase?

git rebase изменяет положение ветки в основной ветке, если есть два комита в освноной ветке, но тестовая ветка находится на первом коммите ее можно переместить на второй коммит с помощью git rebase

что такое git merge?

git merge позволяет соеденить воедино тестовую ветку с основной веткой

что такое git fetch?

git fetch загружает комиты, файлы и ссылки из удаленного репозитория в локальный репозиторий

что такое git pull?

git pull связывает конец тестовой ссылки с концом основной ссылки
